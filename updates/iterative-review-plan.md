# イテレーティブシステムレビュー＆ブラッシュアッププラン

## 概要
サブエージェントを活用した5ラウンド以上のシステムレビューを行い、フィードバックを反映して段階的にアプリを改善する。

各ラウンドで **複数のサブエージェントが並列でレビュー** → **結果を統合** → **優先度付け** → **実装** → **次ラウンドへ** のサイクルを回す。

---

## ラウンド構成

### Round 1: 基盤品質レビュー（コード品質・バグ・セキュリティ）
**目的:** 致命的な問題を最初に潰す

| エージェント | 担当 | レビュー観点 |
|---|---|---|
| Agent A | セキュリティ | APIキーの扱い、XSS、入力バリデーション、localStorage安全性 |
| Agent B | バグ・エラーハンドリング | エッジケース、nullチェック、非同期処理の競合、AbortController |
| Agent C | コード品質 | モジュール設計、責務分離、命名規則、DRY原則 |

**実装:** フィードバックを重要度で分類（Critical / High / Medium）→ Critical & Highを即修正

---

### Round 2: UX/UIレビュー（ユーザビリティ・アクセシビリティ）
**目的:** ユーザー体験の問題を洗い出す

| エージェント | 担当 | レビュー観点 |
|---|---|---|
| Agent D | UXフロー | ステップの分かりやすさ、エラー時の導線、フィードバックの十分さ |
| Agent E | レスポンシブ・モバイル | モバイル対応、タッチ操作、画面サイズ対応 |
| Agent F | アクセシビリティ | aria属性、キーボード操作、コントラスト比、スクリーンリーダー対応 |

**実装:** UX改善（エラーメッセージ・状態遷移）→ レスポンシブ修正 → a11y対応

---

### Round 3: パフォーマンス・最適化レビュー
**目的:** 速度とリソース効率の改善

| エージェント | 担当 | レビュー観点 |
|---|---|---|
| Agent G | API効率 | プロンプトのトークン量、不要なAPI呼び出し、リトライ戦略 |
| Agent H | フロントエンド性能 | DOM操作の効率、画像Base64のメモリ管理、レンダリング最適化 |
| Agent I | ネットワーク・キャッシュ | CDN依存（Tailwind）、キャッシュ戦略、オフライン対応可能性 |

**実装:** トークン削減 → メモリ最適化 → ロード速度改善

---

### Round 4: 機能完成度・エッジケースレビュー
**目的:** Round 1-3の修正後の状態で、機能の抜け漏れを確認

| エージェント | 担当 | レビュー観点 |
|---|---|---|
| Agent J | 機能網羅性 | 各ステップのフロー完全性、履歴復元の正確性、ダウンロード機能 |
| Agent K | エッジケース | 巨大画像、空JSON、API障害時、連続操作、ブラウザ互換性 |
| Agent L | プロンプトエンジニアリング | 分析精度、JSON更新の安定性、画像生成プロンプトの最適化 |

**実装:** 機能補完 → エッジケース対応 → プロンプト改善

---

### Round 5: 総合レビュー（Round 1-4の改善結果を検証）
**目的:** 全修正が正しく機能しているか、新たな問題が生まれていないか最終確認

| エージェント | 担当 | レビュー観点 |
|---|---|---|
| Agent M | リグレッション | Round 1-4の修正が他に影響していないか全体チェック |
| Agent N | ユーザーシナリオ | 実際のユースケースを想定した end-to-end フローの確認 |
| Agent O | コード最終品質 | 最終的なコード品質、一貫性、保守性の確認 |

**実装:** リグレッション修正 → 最終調整 → 完了

---

## 各ラウンドの実行フロー

```
1. サブエージェント起動（並列3体、model: sonnet）
   ↓
2. 各エージェントがレビュー結果をレポート
   ↓
3. メインエージェント（自分）がレポートを統合・優先度付け
   ↓
4. 優先度に基づいて修正を実装（Critical → High → Medium）
   ↓
5. 修正後にコミット
   ↓
6. 次のラウンドへ
```

## ルール
- 各ラウンドで最低3つのサブエージェントを**並列**起動する
- サブエージェントは `model: "sonnet"` を使用（トークン節約）
- 各ラウンド終了時にコミット＆プッシュ
- 各エージェントの指摘は `[Critical/High/Medium/Low]` で重要度を付ける
- Critical・Highは必ず対応、Mediumは時間があれば対応、Lowは記録のみ
